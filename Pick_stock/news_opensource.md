A. 真正免费、开放（最稳）
1) GDELT 2.1（全球新闻数据库）

覆盖：全球新闻（多语言），可用于美股/A股/加密（按关键词、机构名、主题过滤）

优点：真正开放、适合作为“全市场统一新闻底座”

注意：需要你自己做ticker ↔ 公司名/关键词映射与去噪


2) SEC EDGAR（美股“事件新闻”最权威来源之一）

覆盖：美股公司公告/财报/8-K/10-Q/10-K 等“硬事件”

优点：免费、权威、可直接做事件特征（财报/重大事项）

注意：它不是媒体新闻，而是“监管披露”，但对量化更干净；还有访问规范/限流要求


B. 有免费额度（Freemium，但够你项目用）
3) Marketaux（金融新闻 + 情绪）

免费额度：$0/月，100 requests/day（免费层写得很清楚）

覆盖：金融市场新闻（可过滤国家/市场/实体），含情绪/元数据

适用：美股 + 加密 +（也能覆盖部分中国相关新闻）


4) Alpha Vantage – NEWS_SENTIMENT

免费额度：需要 key，免费层有频率/日调用限制（官方有说明）

覆盖：新闻情绪接口，适合快速做“news_score_2h / 24h”

适用：美股/加密都能用（看它对 ticker 的覆盖）


5) Finnhub – Company News

免费计划：$0（个人/开发用途），可拉公司新闻等数据

适用：美股新闻（按公司/市场）


C. 加密“交易所官方公告/动态”（你指定 Binance/Bybit）
6) Bybit Announcements API

Bybit 有官方接口可拉公告/上新/维护等（非常适合做“交易所事件流”特征）


7) Binance 公告/资讯：更现实是 RSS/页面抓取

Binance 有 RSS/资讯入口（很多团队是用 RSS/HTML 抓取做“公告事件流”）

注意：是否有“正式 API”要看 Binance 官方开放情况；工程上常用 RSS 方案


D. A股新闻：完全“官方且免费”的媒体新闻 API 很少见（常用替代）
8) AKShare（开源）

形式：不是传统 SaaS API，而是Python 开源接口库，整合了大量公开数据源（含 A 股相关数据/部分资讯入口）

适用：A股做“能跑起来”的数据与资讯抓取（但要注意各来源网站条款与稳定性）


9) CNINFO（巨潮资讯）WebAPI：偏“官方数据服务平台”（可能需要账号/授权/付费）

优点：公告/披露相关更权威

但它通常不是“随便免费开用”的那种（更像数据服务）


我建议你项目里的“最省事组合”（按市场）

Crypto：Bybit 公告 API + Binance RSS/抓取 + Marketaux（补媒体新闻）

美股：SEC EDGAR（硬事件）+ Finnhub/Marketaux（媒体新闻）+（可选 Alpha Vantage sentiment）

A股：AKShare（抓取/整合）+ Marketaux/GDELT（补媒体新闻）+（若要更权威公告再考虑 CNINFO
